# TASKPACK Step 1A：主角飞船重设计

## 1. 任务目标

将当前占位级玩家机体升级为可作为项目视觉标杆的 `PlayerShipV2`。

本任务不是局部修补，而是完整重构以下四个层面：

- 造型语言
- 材质系统
- 灯光表现
- C1 友好的发光与深度表达

## 2. 当前问题

当前玩家机体存在以下结构性问题：

- 造型仍是 `ConeGeometry + ShapeGeometry + CylinderGeometry` 级别的占位拼装。
- 主体材质仍是 `MeshPhongMaterial`，高光层级不稳定。
- 场景灯光是通用布光，没有“英雄机”专属视觉照明。
- 座舱、喷口、翼面、武装之间缺少明确层级与材质区分。
- 在 C1 裸眼 3D 输出下，体积不足和边缘单薄会被明显放大。

## 3. 设计基准

### 3.1 参考方向

- `Ikaruga`：借鉴清晰轮廓、强对比、读形优先。
- `R-Type Final 2`：借鉴模块化机体、机械分区、表面层次。
- `Radiant Silvergun`：借鉴战斗场景中的主体存在感和主题化配色控制。

### 3.2 目标风格

- 机体气质：高机动、锋利、先进、非写实军武、偏科幻精英战机。
- 轮廓特征：长机鼻、主脊清晰、折角翼、双喷口、腹部武装导轨。
- 视觉特征：银白主装甲、冷色座舱、受控蓝青能量线、橙色喷口核心。

## 4. 范围

### 4.1 必做

- 新建 `PlayerShipV2` 参数化几何生成方案。
- 重构玩家机材质为 PBR 路线。
- 建立主角飞船专属 hero lighting。
- 建立假玻璃方案。
- 建立喷口能量核、导轨灯带、边缘高光方案。
- 为开火、聚焦、受击、自旋技能保留视觉状态接口。

### 4.2 暂不做

- 外部 GLTF 资源导入。
- 真折射玻璃。
- 交织输出后的模糊 bloom。
- 复杂实时阴影系统。

## 5. 技术路线

### 5.1 建模路线

- 使用 `THREE.Group` 组织分件。
- 机体拆分为：
- nose
- spine
- cockpit shell
- left/right wing root
- left/right main wing
- left/right stabilizer
- left/right engine nacelle
- weapon rail
- lower armor

### 5.2 材质路线

- 主装甲：`MeshStandardMaterial`
- 英雄涂层或高等级表面：`MeshPhysicalMaterial`
- 座舱：受控的 `MeshPhysicalMaterial` 或 shader 增强材质
- 能量件：PBR 基底 + emissive
- 需要边缘增强的位置：`onBeforeCompile` 或独立 shader shell

### 5.3 灯光路线

- 保留场景基础灯。
- 增加玩家飞船 hero key light。
- 增加冷色 fill light。
- 增加 rim light，专门强化轮廓边缘。
- 如果成本可控，再补轻量环境反射。

### 5.4 C1 友好路线

- 机体厚度稳定，不依赖纯贴图假深度。
- 关键轮廓用中大尺度面变化表达，不用高频密线。
- 发光点集中，不做全身泛光。
- 玩家机常态深度保持在安全区，爆发技能再上抬。

## 6. 任务卡

### VS-01 造型重构

- 目标：替换现有占位机体，建立新的主体轮廓与分件层级。
- 输出：`PlayerShipV2` 的几何构建代码。
- 验收：静止状态下仅看轮廓即可明显优于旧模型。

### VS-02 PBR 材质工厂

- 目标：建立 hull / canopy / engine / energy rail 的统一材质工厂。
- 输出：可复用材质创建方法。
- 验收：不同部件高光、粗糙度、发光逻辑可独立调参。

### VS-03 Hero Lighting

- 目标：给玩家机建立独立于背景的主光、补光、边缘光策略。
- 输出：玩家机专属灯光接入点。
- 验收：暗背景和复杂背景下机体轮廓仍清晰。

### VS-04 假玻璃与边缘增强

- 目标：实现座舱玻璃与边缘 Fresnel 强化。
- 输出：稳定的假玻璃视觉层。
- 验收：座舱有高级感，但不产生明显排序错乱。

### VS-05 能量与状态反馈

- 目标：将开火、聚焦、自旋、受击接入新的视觉材质参数。
- 输出：状态驱动的材质响应。
- 验收：玩家行为变化可通过机体自身反馈被感知。

### VS-06 C1 稳定性验证

- 目标：确认新方案在多视角交织下不闪、不糊、不脏。
- 输出：验证记录与必要回退项。
- 验收：静止、移动、聚焦、开火、自旋五种状态都可接受。

## 7. 文件边界建议

- 几何与结构：`src/renderer/game/entities/Player.ts`
- 如需拆分：新增 `src/renderer/game/entities/player-ship/`
- 材质与 shader：`src/renderer/rendering/` 下新增对应模块
- 灯光接入：`src/renderer/game/scenes/GameplayScene.ts`
- 配置参数：`src/renderer/game/GameConfig.ts`

## 8. 验收标准

- 主角飞船在静止和移动时都具有明确的三维体积感。
- 新模型明显优于旧模型，不再是占位级几何拼装感。
- 材质不再停留在单一 `Phong` 质感。
- 复杂背景下玩家机依然是第一视觉焦点。
- C1 输出下没有明显发虚、闪烁、串扰加剧。
- 性能没有出现不可接受退化。
